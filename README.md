## <u>Teamate</u> Project Background
- ### 필요성
  - 기존 프레젠테이션 협업 툴들은 실시간 수정 시 다수 사용자의 편집 충돌 발생 시 '최근 변경 내용 덮어쓰기' 방식으로 처리하여 데이터 유실의 위험성이 존재함
  - 또한 기존의 히스토리 기능은 전체 문서 기준으로 복원되기 때문에, 특정 슬라이드만 되돌리고 싶어도 전체 프레젠테이션을 모두 복구해야 하는 비효율이 존재함
  - 때문에 충돌 없는 안정적 동시 편집 환경과 슬라이드 단위의 선택 복원 기능이 요구됨
    
- ### 기존 해결책의 문제점
  - Centralized Lock 기반 방식은 사용자가 많은 경우 확장성이 낮고 충돌 해결 비용이 큼
  - 전체 단위의 Undo/Redo 방식은 실제 협업 환경에서 '부분 상태 복원' 요구를 충족하지 못함
  
## System Design
  - ### System Requirements
    - STOMP 기반 WebSocket 통신을 통한 실시간 편집 동기화
    - CRDT(Yjs) 기반의 충돌 없는 자동 병합 알고리즘 적용
    - 슬라이드 단위 변경 이력 저장 및 특정 시점 선택적 복원 API 제공
    - JWT 인증 및 OAuth2 기반 사용자 인증/권한 관리 지원
    
## Case Study
  - ### Description
    실시간 협업 캔버스 히스토리 및 복원 시스템은 웹 기반 환경에서 다중사용자가 동시에 프레젠테이션을 작성하고 편집할 수 있도록 지원하는 실시간 협업 도구이다.
    STOMP 기반 WebSocket 구조를 통해 사용자 간 편집 내용을 실시간으로 동기화하며, CRDT 알고리즘(Yjs)를 적용해 충돌 없이 안정적인 병합이 가능하다.
    또한 슬라이드 단위의 변경 이력을 관리하고 특정 시점의 슬라이드를 선택적으로 복원할 수 있어, 효율적이고 신뢰성 높은 공동 작업 환경을 제공하는 시스템이다.

  
  
## Conclusion
  - ### 충돌 없는 안정적인 실시간 협업 환경 구현
    - CRDT 기반 자동 병합을 적용함으로써 협업 사용자는 충돌 해결 과정에 신경 쓰지 않고 본인의 작업에 집중할 수 있다.
  - ### 슬라이드 단위의 히스토리 복원 구조 제시
    - 전체 슬라이드 복원 방식이 아닌 슬라이드 개별 복원 구조는 실제 협업 프레젠테이션 제작 상황에서 높은 실무 효용을 보이며, 기존 상용 서비스 대비 차별점을 가진다.
